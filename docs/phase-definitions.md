# Phase definitions

## Fixed baseline

The subgroup chain is generated by successively removing faces from

```text
(U, R, F, L, BR, BL, FR, FL, DR, DL, B, D)
```

and begins, after the initial merged step, as

```text
G0 > G3 > G4 > G5 > G6 > G7 > G8 > G9 > G10 > G'11 > G12.
```

For the four target phases:

| Phase | Quotient transition | Allowed subgroup faces | Upstream coordinate |
|---:|---|---|---|
| 3 | G5 -> G6 | U R F L BR BL FR | `hash7Gen` |
| 4 | G6 -> G7 | U R F L BR BL | `hash6Gen` |
| 5 | G7 -> G8 | U R F L BR | `hash5Gen` |
| 6 | G8 -> G9 | U R F L | `hash4Gen` |

The lookup table for a transition stores the distance from a left coset of the
target subgroup to the target subgroup under the source subgroup's generators.
This identification is checked exactly during full reproduction. The Go table
builder verifies that every target generator and all four nonidentity powers
fix the solved coordinate, while every power of the removed source face is
nontrivial. The independent SymPy audit recomputes the orders of
`G5,G6,G7,G8,G9` from the committed, `FullStateV1`-conjugate actions and checks
that the four consecutive indices are exactly the four table state counts.
Finally, each complete BFS reaches that many coordinate values. Containment of
the target subgroup in the solved fiber plus equality of these indices makes
the target fiber exact; the coordinate cannot be a coarser quotient.

Executable randomized composition tests remain as diagnostics for the coset
side: prepending a target-subgroup word does not change the coordinate, whereas
appending one generally does. These are left cosets `Hs` under the documented
left-to-right token convention. See `state-format.md`.

These values are target-coset eccentricities under the phase source alphabet,
not ordinary Schreier-graph diameters. They are comparison targets only;
regenerated data determines proof coverage. The generated metadata retains the
upstream field name `diameter` for compatibility.

## Physical state

A full physical state has:

- a permutation of 30 edge pieces and 30 edge orientation bits;
- a permutation of 20 corner pieces and 20 corner orientations modulo 3;
- fixed centers under the baseline move set.

The stable representation is the 108-byte `FullStateV1` record documented in
`state-format.md`. It stores piece-at-position arrays; inverse position-of-piece
arrays are derived and checked by the primary implementation.

## Type distinctions

- **Physical state:** a complete reachable Megaminx configuration.
- **Coset:** an equivalence class for one phase transition.
- **Phase index:** the dense integer encoding of a coset.
- **Canonical representative:** a deterministic physical representative chosen
  for a coset or an exact-reduction orbit.

No integer or byte string may be passed between these domains without a typed
adapter and a round-trip test.

## Transition and lifting rule

The second-phase representative lies in the target subgroup of the first
phase, so it is lifted directly as a physical `FullStateV1` element. With left
cosets and left-to-right word replay, a raw pair is reconstructed by starting
at the second representative and then applying the inverse of the stored
first-phase solution. Equivalently, the physical product order is
`representative(second), representative(first)`.

For every raw pair the independent checker verifies all of the following:

1. the resulting state has the requested first phase index;
2. applying the stored optimal first-phase word reaches the exact stored
   second representative;
3. that intermediate state has the requested second phase index;
4. applying the second-phase word reaches solved.

This rule is used identically for phase pairs 3+4 and 5+6; no coordinate
integer is treated as a physical state during lifting.
