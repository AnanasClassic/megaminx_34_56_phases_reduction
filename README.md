# Megaminx phase 3+4 and 5+6 reductions

This repository is the computational companion for the paper
*Exhaustive Boundary-Layer Certification for Megaminx via Neural-Guided Beam
Cascades*.  It proves two exhaustive reductions in the face-turn metric (FTM):

- every maximal-layer composition of phases 3 and 4 has a verified solution
  of length at most 21;
- every maximal-layer composition of phases 5 and 6 has a verified solution
  of length at most 25.

The complete coverage is:

| Pair | Raw compositions | Exact rewrites | Direct certificates | Maximum length |
|---|---:|---:|---:|---:|
| 3+4 | 536,572 | 203 | 536,369 | 21 |
| 5+6 | 407,628 | 8,461 | 399,167 | 25 |

The certificates, not the neural networks, are the proof objects. The
dependency-light Python checker replays every stored word in both the combined
quotient and a full Megaminx simulator. The full reproduction additionally
streams every physical state and canonical word through the separately written
Go verifier. Missing, malformed, over-length, or omitted records make either
path fail closed.

## Claims and dependencies

Let `Omega` be all 48 FTM moves. The local claims are the ambient target bounds
`R_Omega(G5,G7) <= 21` and `R_Omega(G7,G9) <= 25`. Both are proved in the
stronger alphabet `Sigma5` generated by the seven faces of `G5`; for pair 5+6,
the exact-rewrite part may leave the smaller source alphabet `Sigma7`. These
are not claims about ordinary graph diameters. Relative to the published
114-move bound, they give the conditional improvement `114 -> 112`. Together
with the separate color-neutral phase-1 result, they give the conditional
value 111.

This repository does **not** reproduce the large Tomas Rokicki reduction from
116 to 114. The value 114 and the compatibility of the surrounding published
phase chain are explicit external dependencies. See [docs/proof.md](docs/proof.md).

## Fast package audit

Python 3.10+ and XZ support from the Python standard library are sufficient
for the archive audit, proof CLI, and non-neural unit suite:

```bash
make verify-package
make test
```

PyTorch-dependent training/search tests are skipped when the training
dependencies are absent. The exact Schreier--Sims test additionally requires
`requirements-proof.txt`, and cross-language tests that invoke the real Go
binary require a built verifier. Certificate replay itself does not import
PyTorch.

This verifies both compressed and uncompressed certificate hashes, the SQLite
schemas and coverage distributions, and all four checkpoint hashes. It does
not replay the solutions because the regenerated phase tables and compositions
are intentionally not stored in Git.

## Full reproduction

Requirements: Docker, Git, 7-Zip, Python 3.10+, the SymPy proof dependency,
roughly 2 GiB of working disk, and up to ten CPU workers. Install the one
host-side proof dependency with:

```bash
python3 -m pip install -r requirements-proof.txt
```

```bash
make verify-pair34
make verify-pair56
# or both:
make verify-all
```

Each command first recomputes the three relevant subgroup orders, both
individual quotient factors, and the combined target pointwise stabilizer from
the committed problem actions. It checks every action against `FullStateV1`
through the stored conjugacy, without requiring the historical p900 source
file. It then downloads and verifies the pinned upstream archive, builds the
Go table generator and both verifier paths, regenerates the phase tables,
maximal layers, physical Cartesian compositions, and exact reductions, unpacks
the distributed SQLite certificate, and replays 100% of solutions through
Python and Go full-state implementations.
